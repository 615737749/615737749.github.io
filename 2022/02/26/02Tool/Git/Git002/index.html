Git配置多个SSH-Key

发表于 2022-02-26 分类于 02工具

遇到的问题

我在用SSH方式克隆仓库时报错：

git@gitee.com: Permission denied (publickey).
fatal: Could not read from remote repository.

检查电脑上有没有钥匙

打开用户目录 command+shift+g ，输入 ~ 。

显示隐藏文件 command+shift+. 。

检查一下是否有一个叫 .ssh 的文件夹。如果有这个文件夹，检查这个文件夹里是否有钥匙，如果没有就生成钥匙。如果有就拷贝到账户设置里。

生成钥匙

打开终端，输入

ssh-keygen -t rsa -C 'xxx@163.com' -f ~/.ssh/gitee_id_rsa

邮箱和文件名可以自己改。一直回车就可以了，不用输入密码。

ssh-keygen -t rsa -C 'yyy@qq.com' -f ~/.ssh/github_id_rsa

这里演示的是配置多个SSH-key。比方说你想要同时使用gitee和github的仓库。

在本地配置钥匙

在 ~/.ssh 目录下新建一个config文件，添加如下内容（其中Host和HostName填写git服务器的域名，IdentityFile指定私钥的路径）

# gitee
Host gitee.com
HostName gitee.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/gitee_id_rsa
# github
Host github.com
HostName github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/github_id_rsa

在远程配置钥匙

以gitee为例，在浏览器里登陆gitee账号，在设置里有一项SSH公钥，把你刚生成的公钥（gitee_id_rsa.pub）里的内容拷贝到网页里。

测试钥匙

ssh -T git@gitee.com
ssh -T git@github.com
